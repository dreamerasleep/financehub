# 交易紀錄

記錄日常收入與支出。每筆交易會立刻反映到對應帳戶的當前餘額。

## 我能做什麼

- 新增「收入」或「支出」交易
- 編輯、刪除既有交易（餘額自動回滾後再套用新值）
- 用日期區間過濾列表
- 系統預設提供 11 個分類，可直接使用

## 欄位說明

| 欄位 | 必填 | 說明 |
| ---- | ---- | ---- |
| 類型 | ✅ | 「收入」或「支出」 |
| 帳戶 | ✅ | 影響哪個帳戶的餘額 |
| 分類 | ✅ | 須與類型一致：收入分類只能配收入交易 |
| 金額 | ✅ | 正數（最多兩位小數） |
| 日期 | ✅ | 預設為今天 |
| 備註 | ⛔ | 自由文字，最長 255 字元 |

## 操作步驟

1. 從上方導覽列點「交易」
2. 點右上「新增交易」
3. 先選類型（決定下方分類選項）→ 填帳戶 / 分類 / 金額 / 日期 → 儲存

新增成功後，對應帳戶頁的「當前餘額」會立刻變化。

!!! tip "餘額變化規則"
    - 收入 → 帳戶餘額 + 金額
    - 支出 → 帳戶餘額 − 金額
    - 編輯 / 刪除前的舊金額會先還原，再套用新金額

## 預設分類

| 收入 | 支出 |
| ---- | ---- |
| 薪資、獎金、投資收益、其他收入 | 飲食、交通、居住、娛樂、醫療、購物、其他支出 |

目前還不支援自訂分類（規劃中）。

## 常見錯誤訊息

| 訊息 | 原因 |
| ---- | ---- |
| `Category kind does not match transaction type` | 收入交易配了支出分類（或反之） |
| `Account not found` | 帳戶不存在或不屬於本人 |
| `amount: must be positive` | 金額為 0 或負值 |

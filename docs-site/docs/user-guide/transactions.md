# 交易紀錄

記錄日常收入、支出與轉帳。每筆交易會立刻反映到對應帳戶的當前餘額。

## 我能做什麼

- 新增「收入」、「支出」或「轉帳」交易
- 編輯、刪除既有交易（餘額自動回滾後再套用新值）
- 用日期區間過濾列表
- 系統預設提供 11 個分類，可直接使用

## 欄位說明

| 欄位 | 必填 | 說明 |
| ---- | ---- | ---- |
| 類型 | ✅ | 「收入」、「支出」或「轉帳」 |
| 帳戶 | ✅ | 收入 / 支出：影響的帳戶；轉帳：來源帳戶 |
| 目標帳戶 | 轉帳必填 | 僅在類型為「轉帳」顯示；自動過濾為同幣別、不等於來源帳戶 |
| 分類 | 收入 / 支出必填 | 須與類型一致；轉帳不需要分類 |
| 金額 | ✅ | 正數（最多兩位小數） |
| 日期 | ✅ | 預設為今天 |
| 備註 | ⛔ | 自由文字，最長 255 字元 |

## 操作步驟

1. 從上方導覽列點「交易」
2. 點右上「新增交易」
3. 先選類型 → 依類型填餘下欄位：
   - **收入 / 支出**：帳戶 → 分類 → 金額 → 日期
   - **轉帳**：來源帳戶 → 目標帳戶 → 金額 → 日期
4. 儲存

新增成功後，對應帳戶頁的「當前餘額」會立刻變化（轉帳會同時更新兩個帳戶）。

!!! tip "餘額變化規則"
    - 收入 → 帳戶 + 金額
    - 支出 → 帳戶 − 金額
    - 轉帳 → 來源帳戶 − 金額；目標帳戶 + 金額
    - 編輯 / 刪除前的舊金額會先還原（轉帳兩邊同時還原），再套用新金額

## 預設分類

| 收入 | 支出 |
| ---- | ---- |
| 薪資、獎金、投資收益、其他收入 | 飲食、交通、居住、娛樂、醫療、購物、其他支出 |

目前還不支援自訂分類（規劃中）。轉帳不需要分類。

## 常見錯誤訊息

| 訊息 | 原因 |
| ---- | ---- |
| `Category kind does not match transaction type` | 收入交易配了支出分類（或反之） |
| `Transfer source and destination must differ` | 轉帳的來源 / 目標選成同一個帳戶 |
| `Cross-currency transfer not supported` | 來源與目標帳戶幣別不同（規劃中：Sprint 5 匯率上線後解鎖） |
| `Transfer must not have a category` | 轉帳時誤帶 `categoryId` |
| `Account not found` | 帳戶不存在或不屬於本人 |
| `amount: must be positive` | 金額為 0 或負值 |
